# 安全项目模板包使用说明

本模板包用于 ITP4103 Security Applications 小组项目，覆盖：
- Project Proposal
- Phase 1 Report（漏洞评估）
- Phase 2 Final Report（修复方案 + 实施）
- Presentation Script（汇报提纲）

## 使用方式
1. 先改封面信息（组名、成员、日期、项目名）。
2. 先填 Proposal，再复制资产清单/范围到 Phase 1。
3. 完成测试后把截图、请求包、日志填入 Phase 1 证据章节。
4. 在 Phase 2 对每个漏洞填“修复前→修复方案→修复后复测”。
5. 按 Presentation 文稿做 8~10 分钟演示。

## 评分对齐提示
- Security test performed (30%): 至少 3 个高价值漏洞，且有完整复现证据。
- Solution recommended (40%): 每个漏洞给出针对性措施，并尽量有实际代码/配置实施。
- Report (15%): 结构清晰，图表统一。
- Presentation (15%): 演示链路完整，成员分工明确。

## 建议文件命名
- `ITP4103_ProjectProposal_GroupX.pdf`
- `ITP4103_Phase1_GroupX.pdf`
- `ITP4103_Phase2Final_GroupX.pdf`
- `ITP4103_Presentation_GroupX.pptx`
