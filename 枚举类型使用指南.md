# 枚举类型使用指南

本文档说明如何使用新添加的枚举类型替换硬编码的魔法数字。

## 一、可用的枚举类型

### 1. UserType - 用户类型

```python
from app.models import UserType

# 枚举值
UserType.ADMIN        # 0 - 管理员
UserType.TEACHER      # 1 - 教师
UserType.STUDENT      # 2 - 学生

# 获取标签
UserType.ADMIN.label        # '管理员'
UserType.TEACHER.label      # '教师'
UserType.STUDENT.label      # '学生'
```

### 2. QuestionType - 题目类型

```python
from app.models import QuestionType

# 枚举值
QuestionType.CHOICE          # 0 - 选择题
QuestionType.FILL_BLANK      # 1 - 填空题
QuestionType.TRUE_FALSE      # 2 - 判断题
QuestionType.PROGRAMMING     # 3 - 编程题

# 获取标签
QuestionType.CHOICE.label          # '选择题'
QuestionType.FILL_BLANK.label      # '填空题'
QuestionType.TRUE_FALSE.label      # '判断题'
QuestionType.PROGRAMMING.label     # '编程题'
```

### 3. MessageType - 消息类型

```python
from app.models import MessageType

# 枚举值
MessageType.SYSTEM       # 0 - 系统通知
MessageType.PERSONAL     # 1 - 个人消息
MessageType.EXAM         # 2 - 考试通知
MessageType.TASK         # 3 - 任务通知
```

## 二、在代码中使用枚举

### 替换前（硬编码）：

```python
# 检查用户类型
if user.type == 0:
    return BaseView.error('管理员不能参加考试')
elif user.type == 1:
    return BaseView.error('教师不能参加考试')

# 检查题目类型
if question.type == 0:
    score = 2
elif question.type == 3:
    score = 20
```

### 替换后（使用枚举）：

```python
from app.models import UserType, QuestionType

# 检查用户类型
if user.type == UserType.ADMIN:
    return BaseView.error('管理员不能参加考试')
elif user.type == UserType.TEACHER:
    return BaseView.error('教师不能参加考试')

# 检查题目类型
if question.type == QuestionType.CHOICE:
    score = 2
elif question.type == QuestionType.PROGRAMMING:
    score = 20
```

## 三、查询中使用枚举

```python
from app.models import Users, UserType, Practises, QuestionType

# 查询所有管理员
admins = Users.objects.filter(type=UserType.ADMIN)

# 查询所有选择题
choice_questions = Practises.objects.filter(type=QuestionType.CHOICE)

# 排除编程题
non_programming = Practises.objects.exclude(type=QuestionType.PROGRAMMING)
```

## 五、前端使用建议

虽然枚举是Python特性，但前端也可以定义对应的常量：

```javascript
// src/constants/userTypes.js
export const UserType = {
  ADMIN: 0,
  TEACHER: 1,
  STUDENT: 2,

  getLabel(type) {
    const labels = {
      0: '管理员',
      1: '教师',
      2: '学生'
    }
    return labels[type] || '未知'
  }
}
```

## 八、注意事项

1. **数据库兼容性**：
   - 添加枚举不会影响现有数据
   - 数据库中的整数值保持不变
   - choices参数只影响Django Admin和表单验证

2. **性能**：
   - 枚举访问是常量时间，无性能损失
   - 比字符串比较更快

3. **向后兼容**：
   - 硬编码的数字仍然有效
   - 可以逐步迁移，不需要一次性全部更改

4. **类型提示**：
   ```python
   def get_students(user_type: UserType) -> list:
       return Users.objects.filter(type=user_type)
   ```
