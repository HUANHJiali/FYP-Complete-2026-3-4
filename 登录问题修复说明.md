# 前端登录问题 - 已修复

## 问题原因

前端代理无法连接到后端服务器（ECONNREFUSED）

## 解决方案

已修改 `source/client/src/utils/http.js`，让前端直接连接后端而不通过代理。

**修改内容**:
- 开发环境: `http://127.0.0.1:8000/api`
- 不再使用vue.config.js中的代理配置

## 测试步骤

### 1. 访问前端
打开浏览器，访问: **http://localhost:8080**

### 2. 测试登录
使用以下任一账号：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | 123456 | 管理员 |
| teacher | 123456 | 教师 |
| student | 123456 | 学生 |

### 3. 如果还是不行

**检查浏览器控制台** (按F12):

**Console标签**:
- 查看是否有CORS错误
- 查看是否有网络错误
- 截图所有红色错误

**Network标签**:
1. 刷新页面
2. 尝试登录
3. 找到login请求
4. 点击查看：
   - Request URL（是否是 http://127.0.0.1:8000/api/login/）
   - Status Code（200? 404? 其他?）
   - Response内容

### 4. 可能的CORS问题

如果看到CORS错误，后端需要允许来自localhost:8080的请求。

**临时解决方案**: 使用Chrome插件 Allow CORS

### 5. 终极方案

如果前端还是不行，可以：

**直接访问后端API**:
```bash
# 测试登录API
curl -X POST http://127.0.0.1:8000/api/login/ \
  -H "Content-Type: application/json" \
  -d '{"userName":"admin","passWord":"123456"}'
```

这应该会返回成功并给出token。
