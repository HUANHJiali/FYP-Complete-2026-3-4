# 部署演示测试资料包（2026-03-01）

本资料用于“部署后快速证明系统可演示”，按最小步骤完成可重复验证。

## 1. 文件清单
- `demo_quick_check.ps1`：一键执行容器检查 + 全功能冒烟 + 考试链路专项回归
- `DEMO_CHECKLIST_20260301.md`：现场演示步骤与预期结果清单
- `FEATURE_SMOKE_REPORT_20260301.md`：最近一次自动化结果留档（含重启后专项回归）
- `答辩口播稿_30秒_20260301.md`：快速开场版口播稿
- `答辩口播稿_3分钟_20260301.md`：完整演示版口播稿

## 2. 推荐演示流程（约 3 分钟技术证明 + 8 分钟功能演示）
1) 在项目根目录执行：
   - `powershell -ExecutionPolicy Bypass -File .\demo_quick_check.ps1`
2) 展示自动化结果：
   - 全功能冒烟 `25/25` 通过
   - 考试链路专项回归 `3/3` 通过
3) 按 `DEMO_CHECKLIST_20260301.md` 完成三角色业务演示

## 3. 默认演示账号
- 管理员：`admin / 123456`
- 教师：`teacher / 123456`
- 学生：`student / 123456`

## 4. 关键证明点（建议重点口播）
- 学生交卷后不再卡顿超时，提交链路稳定
- 提交后状态进入“待发布”，教师端可执行发布
- 发布后学生可查看结果，形成考试业务闭环
- AI 评分与 AI 出题接口可正常调用

## 5. 可选初始化（如需补充练习数据）
在后端容器中执行：
- `docker exec -it fyp_backend python init_practice_data.py`

> 说明：若现场不希望改动已有数据，可跳过此步骤。
