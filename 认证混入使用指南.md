# 认证混入使用指南

## 一、概述

`authMixin.js` 提供了统一的认证相关方法，可以避免在13个组件中重复编写token获取逻辑。

## 二、功能特性

### 1. 自动获取token
- 优先从Vuex store获取
- 如果store中没有，从sessionStorage获取
- 通过computed属性自动响应式更新

### 2. 便捷的登录检查
- 一行代码检查登录状态
- 未登录时自动跳转到登录页
- 可自定义提示消息

### 3. 统一的token管理
- 保存token到store和sessionStorage
- 清除登录状态
- 构建带token的请求参数

## 三、使用方法

### 基本用法

```javascript
// 在组件中引入混入
import authMixin from '@/mixins/authMixin'

export default {
  mixins: [authMixin],

  methods: {
    loadData() {
      // 使用this.token替代原来的写法
      // 旧的: this.$store.state.token || sessionStorage.getItem('token')
      // 新的: this.token
      getLoginUser(this.token).then(resp => {
        // ...
      })
    }
  }
}
```

### 完整示例

```vue
<template>
  <div>
    <Button @click="handleStart" v-if="isLoggedIn">
      开始考试
    </Button>
  </div>
</template>

<script>
import authMixin from '@/mixins/authMixin'
import { getLoginUser } from '@/api/index'

export default {
  mixins: [authMixin],

  data() {
    return {
      userInfo: {}
    }
  },

  mounted() {
    // 检查登录状态
    if (this.checkLogin()) {
      this.loadUserInfo()
    }
  },

  methods: {
    loadUserInfo() {
      // 使用this.token
      getLoginUser(this.token).then(resp => {
        this.userInfo = resp.data
      })
    },

    handleStart() {
      // 检查登录
      if (!this.checkLogin('请先登录后再开始考试')) {
        return
      }

      // 使用getAuthParams构建请求参数
      const params = this.getAuthParams({
        examId: this.$route.query.id
      })

      // 发起请求
      addExamLog(params).then(resp => {
        // ...
      })
    }
  }
}
</script>
```

## 四、API参考

### Computed属性

#### `token`
- **类型**: String
- **说明**: 当前用户的token，优先从store获取，其次从sessionStorage获取

#### `isLoggedIn`
- **类型**: Boolean
- **说明**: 是否已登录（token非空）

### Methods

#### `checkLogin(message)`
- **参数**:
  - `message` (String): 提示消息，默认为'请先登录'
- **返回值**: Boolean - 是否已登录
- **说明**: 检查登录状态，未登录时显示提示并跳转登录页

#### `clearAuth()`
- **返回值**: 无
- **说明**: 清除登录状态（store和sessionStorage）

#### `saveToken(token)`
- **参数**:
  - `token` (String): token字符串
- **返回值**: 无
- **说明**: 保存token到store和sessionStorage

#### `getAuthParams(params)`
- **参数**:
  - `params` (Object): 原始参数对象
- **返回值**: Object - 包含token的参数对象
- **说明**: 构建带token的请求参数

## 九、注意事项

1. **不要在data中定义token**: token是computed属性，会自动更新
2. **优先使用checkLogin**: 比手动检查this.token更友好
3. **保存token使用saveToken**: 确保store和sessionStorage同步
4. **清除登录使用clearAuth**: 一次性清除所有认证信息

## 十、常见问题

### Q: 迁移后token为空？
A: 检查Vuex store是否正确配置了SET_TOKEN mutation

### Q: checkLogin没有跳转？
A: 确保$this.$router已正确配置

### Q: 如何在非组件文件中使用？
A: 可以直接使用：
```javascript
import store from '@/store'
const token = store.state.token || sessionStorage.getItem('token')
```
