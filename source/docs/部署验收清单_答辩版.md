# 部署验收清单（答辩版）

## 目标
确保演示环境与可交付环境具备：
- 可启动
- 可登录
- 关键主链路可用
- 安全基线已启用

## 一、环境变量核对

```env
DEBUG=False
SECRET_KEY=高强度随机串
ALLOWED_HOSTS=你的域名,你的IP
CORS_ALLOWED_ORIGINS=https://你的前端域名
CSRF_TRUSTED_ORIGINS=https://你的前端域名
CSRF_EXEMPT_PATH_PREFIXES=/api/
ENABLE_RATE_LIMIT=True
RATELIMIT_ENABLE=True
TRUST_PROXY_HEADERS=False
ENABLE_PERFORMANCE_MONITOR=True
ENABLE_DB_QUERY_OBSERVE=False
SLOW_REQUEST_THRESHOLD_MS=1000
SLOW_DB_QUERY_THRESHOLD_MS=200
SESSION_COOKIE_SECURE=True
CSRF_COOKIE_SECURE=True
```

## 二、启动与基础检查

```bash
cd source/server
python manage.py check
python manage.py migrate
python manage.py runserver
```

验收点：
- `check` 无报错
- 服务启动成功
- `GET /api/health/` 返回健康状态
- 性能日志可输出（`logs/exam.log` 或控制台）

## 三、四条主链路回归

```bash
cd source/server
python tools/run_regression_baseline.py
```

验收点：
- 登录链路通过
- 考试分页链路通过
- 练习分页链路通过
- AI评分接口链路通过

## 三点五、一键答辩校验（推荐）

```bash
cd source/server
python tools/run_defense_demo_check.py
```

Windows 双击方式：

```bat
source\server\tools\run_defense_demo_check.bat
```

Linux/Mac 快捷方式：

```bash
cd source/server
bash tools/run_defense_demo_check.sh
```

该命令会自动执行：
- `manage.py check`
- 四链路回归基线
- 关键健康接口检查（`/api/health/simple/`、`/api/health/`）

说明：
- 默认模式下，`/api/health/simple/` 必须通过；`/api/health/` 在数据库/外部服务未就绪时仅告警。
- 若需要“生产级严格校验”，可执行：

```bash
STRICT_HEALTH=1 python tools/run_defense_demo_check.py
```

## 五、性能与稳定性（本轮）

- 已对高频分页接口启用短时响应缓存（60秒）：
  - `/api/exams/page/`
  - `/api/practicepapers/page/`

验收建议：
- 连续刷新列表页，响应时间应更稳定。
- 压测/联调环境可临时开启 `ENABLE_DB_QUERY_OBSERVE=True`，定位慢 SQL 后恢复为 `False`。

## 六、风险项与处置

1. 数据库未就绪：`/api/health/` 可能非 200；先确认 DB，再执行严格校验。
2. AI Key 缺失：AI评分链路失败；演示前用 `tools/diagnose_zhipuai.py` 预检。
3. CORS/CSRF 配置错误：前端跨域请求失败；对照 `.env` 白名单修复。
4. 观测开关误用：`ENABLE_DB_QUERY_OBSERVE=True` 会增加开销，仅排障时开启。
5. 代理头信任误配：未在可信反向代理后却开启 `TRUST_PROXY_HEADERS=True` 可能导致限流IP被伪造。

## 七、回滚方案（最小）

若出现异常：
1. 回滚本次改动文件
2. 保留 `DEBUG=True` 在本地排查
3. 重新执行：

```bash
python manage.py check
python tools/run_regression_baseline.py
```
