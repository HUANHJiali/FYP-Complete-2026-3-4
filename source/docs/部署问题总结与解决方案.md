# FYP 项目 AWS 部署问题总结与解决方案

## 问题清单

### 1. **AMI ID 无效**
- **问题**: `InvalidAMIID.NotFound: The image id '[ami-xxx]' does not exist`
- **原因**: AMI ID 在指定区域不存在或已过期
- **解决方案**: 使用有效的 Ubuntu 22.04 LTS AMI ID (`ami-0ecb62995f68bb549`)

### 2. **AWS 凭证过期**
- **问题**: `InvalidClientTokenId: The security token included in the request is invalid`
- **原因**: AWS Academy Learner Lab 凭证是临时的，会过期
- **解决方案**: 定期从 AWS Academy 门户重新获取凭证并更新环境变量

### 3. **操作系统不匹配**
- **问题**: `sudo: docker: command not found`, `apt-get: command not found`
- **原因**: EC2 实例是 Amazon Linux，但脚本使用 Ubuntu 命令
- **解决方案**: 使用 Ubuntu 22.04 LTS AMI

### 4. **GitHub 仓库访问失败**
- **问题**: `fatal: could not read Username for 'https://github.com': No such device or address`
- **原因**: 仓库是私有的，无法在 user_data 中自动克隆
- **解决方案**: 将仓库设为公开，或使用 Personal Access Token

### 5. **Docker 构建路径错误**
- **问题**: `ERROR: build path /home/ubuntu/25FYP/deploy/source/server either does not exist`
- **原因**: 脚本在错误的工作目录执行，相对路径不正确
- **解决方案**: 在脚本开头 `cd /home/ubuntu/25FYP` 确保在项目根目录

### 6. **Nginx 配置文件找不到**
- **问题**: `COPY failed: file not found in build context or excluded by .dockerignore: stat docker/frontend/nginx.conf: file does not exist`
- **原因**: Dockerfile 构建上下文与文件路径不匹配
- **解决方案**: 
  - 将 `nginx.conf` 复制到 `source/client/nginx.conf`
  - 使用 volume 挂载或 Dockerfile 内联创建配置

### 7. **CORS 配置错误**
- **问题**: `Origin 'http://<?xml version="1.0" encoding="iso-8859-1"?>' in CORS_ALLOWED_ORIGINS should not have query`
- **原因**: `.env` 文件中的 `CORS_ALLOWED_ORIGINS` 包含无效的 XML 内容（metadata 服务返回错误）
- **解决方案**: 在部署脚本中添加错误处理，使用 `localhost` 作为后备值

### 8. **Django Migration 冲突**
- **问题**: `django.db.utils.OperationalError: (1050, "Table 'fater_practice_papers' already exists")`
- **原因**: 数据库表已存在，但 Django migration 记录显示未应用
- **解决方案**: 
  - 清空数据库重新迁移（演示环境）
  - 或使用 `migrate --fake-initial` 标记已存在的表

### 9. **Gunicorn 未安装**
- **问题**: `bash: line 1: gunicorn: command not found`
- **原因**: `requirements.txt` 中缺少 `gunicorn`
- **解决方案**: 在 `requirements.txt` 中添加 `gunicorn==21.2.0`

### 10. **mysqlclient 编译慢**
- **问题**: `Building wheel for mysqlclient (setup.py): started` 卡住很久
- **原因**: `mysqlclient` 需要编译 C 扩展，在资源有限的 EC2 上很慢
- **解决方案**: 移除 `mysqlclient`，只使用 `pymysql`（项目已配置支持）

## 自动化解决方案

所有这些问题已通过以下方式自动处理：

1. **Terraform 配置** (`infrastructure/terraform/main.tf`)
   - 使用正确的 Ubuntu AMI
   - 自动安装 Docker 和依赖
   - 自动克隆/更新代码仓库
   - 后台运行部署脚本避免超时

2. **部署脚本** (`deploy/aws-deploy.sh`)
   - 自动检测并修复 EC2 IP 获取失败
   - 自动添加 `gunicorn` 到 `requirements.txt`
   - 自动移除 `mysqlclient`（避免编译慢）
   - 自动处理 CORS 配置错误
   - 自动创建交换空间（内存不足时）
   - 自动处理数据库迁移冲突

3. **Docker Compose 配置** (`docker-compose.prod.yml`)
   - 正确的构建上下文
   - 正确的端口映射（允许 ALB 访问）
   - 健康检查配置

## 使用说明

### 部署步骤

1. **配置 AWS 凭证**
   ```powershell
   cd infrastructure/terraform
   .\set-env.ps1
   ```

2. **初始化 Terraform**
   ```bash
   terraform init
   ```

3. **部署基础设施**
   ```bash
   terraform apply
   ```

4. **查看部署日志**（SSH 到 EC2）
   ```bash
   tail -f /tmp/deploy.log
   ```

### 验证部署

```bash
# 获取 ALB DNS 名称
terraform output alb_dns_name

# 测试前端
curl http://<ALB_DNS>/

# 测试后端 API
curl http://<ALB_DNS>/api/projects/all/
```

## 故障排查

### 查看容器状态
```bash
docker-compose -f docker-compose.prod.yml ps
```

### 查看日志
```bash
# 所有服务
docker-compose -f docker-compose.prod.yml logs -f

# 特定服务
docker-compose -f docker-compose.prod.yml logs -f backend
```

### 修复 Migration 冲突
```bash
# 方法 1: 清空数据库（演示环境）
docker-compose -f docker-compose.prod.yml exec db mysql -uroot -pExam123456! -e "DROP DATABASE IF EXISTS db_exam; CREATE DATABASE db_exam CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
docker-compose -f docker-compose.prod.yml restart backend

# 方法 2: Fake 初始迁移
docker-compose -f docker-compose.prod.yml exec backend python manage.py migrate --fake-initial
```

### 重新构建镜像
```bash
docker-compose -f docker-compose.prod.yml build --no-cache backend
docker-compose -f docker-compose.prod.yml up -d backend
```
