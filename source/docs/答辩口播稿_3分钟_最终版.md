# 答辩口播稿（3分钟最终版）

各位老师好，我汇报的项目是智能学生考试系统。系统采用 Django + Vue 架构，面向管理员、教师、学生三类用户，支持题库管理、考试管理、练习试卷、错题分析以及 AI 评分。

我这次工作的重点不是堆功能，而是把系统做到“可交付”。主要分为三部分。

## 一、稳定可运行
我先保证系统在答辩环境下可以稳定启动和验证。现在通过一条命令就可以自动执行：
- Django 配置检查
- 四条主链路回归（登录、考试、练习、AI评分）
- 健康接口检查
这样每次改动后都能快速确认没有回归，避免“改一处坏一片”。

## 二、最小安全基线
考虑到 FYP 场景，我采用了“高收益、低复杂度”的安全策略。
- 配置层面：关键项环境变量化
- CSRF 策略：从全局豁免改为 API 白名单豁免
- 限流策略：可配置并默认开启
- 文档层面：补齐了上线前核查清单

这样做的价值是：既不增加过多复杂度，也能体现工程风险意识和可解释性。

## 三、低风险性能优化
我对高频分页接口做了 60 秒短缓存，主要是考试列表和练习试卷列表。
这个优化很克制，目标不是追求极限性能，而是提高答辩和演示中的响应稳定性，减少重复查询抖动。

## 四、交付成果
目前我已经形成了完整交付包：
- 安全基线清单
- 部署验收清单
- 一键演示校验脚本
- 5分钟演示顺序卡
- 高频追问 Q&A

所以我这次 FYP 的核心价值是：不仅实现了业务功能，还建立了可验证、可复现、可演示的工程闭环。

如果老师允许，我可以现场直接运行一键脚本，展示完整校验流程。谢谢老师。
