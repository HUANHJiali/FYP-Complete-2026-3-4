# 答辩演示顺序卡（5分钟版）

> 总入口：`答辩资料总索引_最终版.md`

## 0:00 - 0:30 开场

建议话术：
- 本系统是基于 Django + Vue 的智能考试平台，覆盖管理员、教师、学生三角色。
- 本次重点展示三件事：稳定性、可回归、可交付。

## 0:30 - 1:20 一键校验（稳定性）

在终端执行：

```bat
tools\run_defense_demo_check.bat
```

建议话术：
- 这条命令会自动跑 Django 配置检查、四链路回归和健康检查。
- 输出 PASS 说明当前版本可稳定演示。

## 1:20 - 2:40 功能主链路（业务价值）

按顺序展示：
1. 管理员登录并进入管理后台
2. 教师端查看考试/练习列表（分页加载）
3. 学生端进入练习列表并查看状态
4. AI 评分接口返回结果结构

建议话术：
- 四条链路分别对应：登录、考试、练习、AI评分。
- 这四条是我定义的“最小可交付回归基线”。

## 2:40 - 3:40 工程完善（可维护性）

建议展示文档：
- 部署验收清单
- P0 安全基线清单

建议话术：
- 我没有做过度工程，而是做了 FYP 场景的最小安全基线。
- 关键点是：环境变量化、CSRF白名单豁免、限流默认开启。
- 新增可复跑脚本，后续每次改动都能快速验证。

## 3:40 - 4:30 性能优化（实效）

建议话术：
- 对高频分页接口加了 60 秒短缓存：考试列表和练习试卷列表。
- 目标是低风险提升稳定性，避免重复查询抖动。

## 4:30 - 5:00 收尾

建议话术：
- 当前版本已具备：可部署、可回归、可答辩演示。
- 后续计划是继续做视图拆分与慢查询观测，逐步演进，不影响现有功能。

## 现场异常应对（备用）

### 情况1：完整健康检查返回500
- 原因：本地未连接MySQL/未配置AI Key
- 处理：说明当前脚本默认非严格模式，`/api/health/simple/` 已通过，主链路回归已通过

### 情况2：前端页面偶发加载慢
- 处理：刷新后重试，并说明已加短缓存稳定列表查询

### 情况3：老师追问“为什么不做更重的安全”
- 回答：按 FYP 目标做最小安全基线，优先确保功能完整、验证可复现、风险可解释
