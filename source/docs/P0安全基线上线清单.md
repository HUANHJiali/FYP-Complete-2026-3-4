# P0 安全基线上线清单（答辩版）

## 目标
在不影响当前 FYP 功能演示的前提下，完成最小安全基线收敛：
- 配置可控
- 风险可解释
- 上线可复现

## 本次已完成改动

### 1) 密钥与调试策略
- `SECRET_KEY`：生产环境（`DEBUG=False`）下必须由环境变量提供。
- `DEBUG`：默认 `True`（保证本地开箱可跑），但文档与清单明确要求生产显式设为 `False`。

### 2) CSRF 策略统一
- 移除 `BaseView` 全局 `csrf_exempt`。
- 新增路径白名单中间件：仅对白名单路径（默认 `/api/`）豁免 CSRF。
- 保留 `CsrfViewMiddleware`，使 Django Admin 等非 API 页面仍受 CSRF 保护。

### 3) CORS 默认行为收敛
- 若配置 `CORS_ALLOWED_ORIGINS`：按白名单放行。
- 未配置时：仅开发模式(`DEBUG=True`)允许全放开，非开发模式默认不放开。

### 4) 限流策略
- 新增 `ENABLE_RATE_LIMIT` 开关，默认启用通用限流中间件。
- 登录接口已有 `ratelimit` 装饰器，形成双层保护。

### 5) 基础安全头
- 开启 `SECURE_CONTENT_TYPE_NOSNIFF`
- 设置 `X_FRAME_OPTIONS=DENY`
- `SESSION_COOKIE_SECURE`/`CSRF_COOKIE_SECURE` 支持环境变量控制

## 关键配置（.env）

```env
CORS_ALLOWED_ORIGINS=https://你的前端域名
CSRF_TRUSTED_ORIGINS=https://你的前端域名

# 本次新增
CSRF_EXEMPT_PATH_PREFIXES=/api/
ENABLE_RATE_LIMIT=True
RATELIMIT_ENABLE=True
TRUST_PROXY_HEADERS=False
SESSION_COOKIE_SECURE=True
CSRF_COOKIE_SECURE=True
```

## 增量收敛记录（2026-02-28）

### A) 管理端模块化与重复逻辑收口
- 已将 Admin 关键域统一到独立模块并保持双路径（legacy/split）行为一致：
	- dashboard：`admin_dashboard_views.py`
	- statistics：`admin_statistics_views.py`
	- export/import：`admin_export_views.py`
	- tasks：`admin_task_views.py`
- legacy `views.py` 对应入口已改为委托，减少大段重复业务代码。

### B) 可证伪死代码清理（零行为变更）
- 删除了可确认“无分发入口 + 无外部引用”的包装方法与冗余分支。
- 清理了多个未使用变量/导入与不可达分支，降低维护噪音与误改风险。

### C) 时区风险收敛
- `admin_dashboard_views.py` 已从 `datetime.now()` 与字符串时间过滤，统一为 `timezone.now()` + aware datetime 比较。
- 目标是消除 `USE_TZ` 下的 naive datetime 告警，避免后续线上时区隐患。

### D) 回归验证结论
- 每个切片均执行：
	- `python manage.py check`
	- 对应域的定向测试（dashboard/statistics/tasks/messages/question-ai 等）
	- `python tools/run_regression_baseline.py`
- 当前阶段结果：上述检查持续通过，未出现功能回归。

### F) 测试稳定性修复（本轮新增）
- 发现批量执行管理端测试时，`/api/login/` 触发限流导致部分 `setUp` 403（非业务功能问题）。
- 已在 `settings.py` 增加测试态默认策略：
	- `ENABLE_RATE_LIMIT` 在测试进程默认关闭
	- `RATELIMIT_ENABLE` 在测试进程默认关闭
- 为减少测试噪音与无关开销，测试进程默认关闭性能监控中间件：
	- `ENABLE_PERFORMANCE_MONITOR` 在测试进程默认关闭
- 生产/开发仍可通过环境变量显式控制，不影响线上限流策略。

### G) 限流规则命中修复（本轮新增）
- 修复了 `RateLimitMiddleware` 路径解析逻辑：`/api/admin/*` 现在会正确命中 `/api/admin/` 前缀规则。
- 采用“精确匹配优先 + 最长前缀匹配 + 默认值兜底”的策略，避免规则被静默跳过。
- 已新增单元测试覆盖：
	- 精确路径（`/api/login/`）
	- 管理前缀（`/api/admin/users/`）
	- 默认兜底（未知路径）

### H) 客户端 IP 识别加固（本轮新增）
- `RateLimitMiddleware.get_client_ip` 新增代理场景兼容：
	- 仅在显式信任代理头时解析 `X-Forwarded-For`（取首个有效 IP）
	- 仅在显式信任代理头时解析 `X-Real-IP`
	- 最后 `REMOTE_ADDR`，无值时返回 `unknown`
- 同步清理中间件未使用导入，减少维护噪音。
- 已补充单元测试覆盖以上 4 种来源分支，保证后续改动可回归。

### I) 代理头信任边界（本轮新增）
- 新增配置：`TRUST_PROXY_HEADERS`（默认 `False`）。
- 默认不信任代理头，避免客户端伪造 `X-Forwarded-For/X-Real-IP` 影响限流判定。
- 仅在明确部署于可信反向代理（Nginx/Ingress）并做好源地址保护时再开启。

### J) Docker 编排对齐（本轮新增）
- 已在 `docker-compose.yml` 与 `docker-compose.prod.yml` 的 backend 环境变量中补齐：
	- `CSRF_EXEMPT_PATH_PREFIXES`
	- `ENABLE_RATE_LIMIT`
	- `RATELIMIT_ENABLE`
	- `TRUST_PROXY_HEADERS`
- 已同步补齐 `dockerpull/docker-compose.yml` 对应 backend 环境变量，避免另一套拉取镜像编排漂移。
- 目标：保证“文档、模板、编排文件”三处配置一致，减少部署遗漏。
