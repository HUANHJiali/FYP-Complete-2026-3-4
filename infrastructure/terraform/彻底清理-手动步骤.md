# 彻底清理 Git 历史中的敏感信息 - 手动步骤

## 问题

提交 `17b7581` 中包含 AWS 凭证，GitHub 拒绝推送。需要从历史中移除这个提交或修改它。

## 解决方案：使用交互式 Rebase

### 步骤 1: 找到问题提交的父提交

```powershell
git log --oneline
# 找到 17b7581 之前的提交：46e3389
```

### 步骤 2: 开始交互式 Rebase

```powershell
git rebase -i 46e3389
```

### 步骤 3: 在编辑器中修改

会打开一个编辑器，显示：
```
pick 17b7581 修复502错误：添加健康检查端点并更新ALB配置
```

**修改为**：
```
edit 17b7581 修复502错误：添加健康检查端点并更新ALB配置
```

保存并关闭编辑器。

### 步骤 4: 修改提交

Git 会停在那个提交，然后：

```powershell
# 移除包含敏感信息的文件
git rm infrastructure/terraform/set-env.ps1

# 或者，如果文件已经不存在，直接继续
# 修改提交（移除文件）
git commit --amend --no-edit

# 继续 rebase
git rebase --continue
```

### 步骤 5: 验证

```powershell
# 检查提交中是否还有敏感信息
git show HEAD:infrastructure/terraform/set-env.ps1

# 应该显示 "fatal: path 'infrastructure/terraform/set-env.ps1' does not exist"
# 或者显示新版本（不包含敏感信息）
```

### 步骤 6: 强制推送

```powershell
git push origin --force main
```

## 如果 Rebase 失败

如果遇到问题，可以中止：

```powershell
git rebase --abort
```

## 替代方案：使用 Git Filter-Repo（如果已安装）

```powershell
# 安装 git-filter-repo
pip install git-filter-repo

# 从历史中移除文件
git filter-repo --path infrastructure/terraform/set-env.ps1 --invert-paths

# 强制推送
git push origin --force --all
```

## 最简单的方案：创建新分支

如果历史不重要：

```powershell
# 1. 创建新分支
git checkout --orphan clean-main

# 2. 添加所有文件（除了 set-env.ps1）
git add .
git reset HEAD infrastructure/terraform/set-env.ps1

# 3. 提交
git commit -m "Initial commit - cleaned history"

# 4. 删除旧分支并重命名
git branch -D main
git branch -m main

# 5. 强制推送
git push origin --force main
```

## 推荐步骤（最简单）

**如果这是你的个人仓库，且历史不重要，使用方法 3（创建新分支）最简单。**
