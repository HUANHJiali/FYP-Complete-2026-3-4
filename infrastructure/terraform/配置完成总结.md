## 📋 使用步骤

### 1. 设置 AWS 凭证

**方法 A: 使用脚本（推荐）**
```powershell
cd infrastructure/terraform
.\set-env.ps1
```

**方法 B: 手动设置**
```powershell
# 从 AWS Academy Learner Lab 获取凭证后，替换下面的占位符
$env:AWS_ACCESS_KEY_ID="YOUR_ACCESS_KEY_ID_HERE"
$env:AWS_SECRET_ACCESS_KEY="YOUR_SECRET_ACCESS_KEY_HERE"
$env:AWS_SESSION_TOKEN="YOUR_SESSION_TOKEN_HERE"
$env:AWS_DEFAULT_REGION="us-east-1"
```

### 2. 准备 SSH 密钥

确保有 SSH 公钥文件，或修改 `terraform.tfvars` 中的 `public_key_content`。

### 3. 预览部署

```powershell
terraform plan
```

### 4. 实际部署

```powershell
terraform apply
```

输入 `yes` 确认。

## ⚠️ 重要提示

### 凭证有效期

AWS Academy Learner Lab 的凭证是**临时的**，通常有效期为 1-4 小时。

如果遇到 `InvalidClientTokenId` 错误：
1. 重新登录 AWS Academy
2. 启动新的 Learner Lab
3. 获取新的凭证
4. 更新环境变量

### 凭证更新

当凭证过期时，需要：
1. 从 Learner Lab 获取新凭证
2. 更新 `set-env.ps1` 脚本中的凭证
3. 或直接更新环境变量
