# 修复部署问题指南

## 当前问题

1. **前端构建失败** - nginx.conf 文件路径问题
2. **EC2 IP 获取失败** - 返回 401 Unauthorized
3. **Docker 容器未启动** - 因为构建失败

## 快速修复步骤

### 在 EC2 上执行以下命令：

```bash
cd /home/ubuntu/25FYP

# 1. 确保 nginx.conf 文件存在
ls -la source/client/nginx.conf

# 如果不存在，创建它
cat > source/client/nginx.conf << 'EOF'
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态文件缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
EOF

# 2. 修复 .env 文件（移除特殊字符）
cd /home/ubuntu/25FYP
cat > .env << 'ENVEOF'
DB_HOST=db
DB_PORT=3306
DB_NAME=db_exam
DB_USER=examuser
DB_PASSWORD=Exam123456!
MYSQL_ROOT_PASSWORD=Exam123456!
SECRET_KEY=$(openssl rand -base64 32)
DEBUG=False
ALLOWED_HOSTS=localhost,127.0.0.1,*
CORS_ALLOWED_ORIGINS=http://localhost,http://*
ENVEOF

# 3. 重新构建并启动
docker-compose -f docker-compose.prod.yml down
docker-compose -f docker-compose.prod.yml build --no-cache
docker-compose -f docker-compose.prod.yml up -d

# 4. 查看日志
docker-compose -f docker-compose.prod.yml logs -f
```

## 如果问题仍然存在

### 检查文件权限

```bash
chmod 644 source/client/nginx.conf
```

### 手动测试构建

```bash
cd source/client
docker build -f ../../docker/frontend/Dockerfile.prod -t test-frontend .
```

### 查看详细错误

```bash
docker-compose -f docker-compose.prod.yml build frontend 2>&1 | tee /tmp/build.log
```
