# 修复 502 Bad Gateway 错误 - 快速步骤

## 问题原因

502 错误表示 ALB 无法连接到后端服务，可能的原因：
1. 后端服务未启动
2. 健康检查失败
3. 端口未正确监听
4. 安全组配置问题

## 快速修复步骤

### 步骤 1: SSH 到 EC2 实例

```bash
# 使用 Terraform 输出的 SSH 命令
ssh -i fyp-keypair.pem ubuntu@<EC2_IP>
```

### 步骤 2: 运行诊断脚本

```bash
cd /home/ubuntu/25FYP
chmod +x infrastructure/terraform/诊断502错误.sh
bash infrastructure/terraform/诊断502错误.sh
```

或者手动检查：

```bash
# 1. 检查容器状态
docker-compose -f docker-compose.prod.yml ps

# 2. 检查端口监听
sudo ss -tlnp | grep -E ':(80|8000)'

# 3. 测试本地服务
curl http://localhost/
curl http://localhost:8000/api/health/

# 4. 查看日志
docker-compose -f docker-compose.prod.yml logs --tail=50 backend
```

### 步骤 3: 如果服务未运行，手动启动

```bash
cd /home/ubuntu/25FYP

# 停止所有容器
docker-compose -f docker-compose.prod.yml down

# 重新构建并启动
docker-compose -f docker-compose.prod.yml build --no-cache backend
docker-compose -f docker-compose.prod.yml up -d

# 等待服务启动
sleep 20

# 检查状态
docker-compose -f docker-compose.prod.yml ps
```

### 步骤 4: 检查健康检查端点

```bash
# 测试健康检查
curl http://localhost:8000/api/health/

# 应该返回 JSON:
# {"status":"healthy","checks":{...},"timestamp":"..."}
```

### 步骤 5: 检查 ALB 目标组健康状态

在 AWS Console 中：
1. 进入 **EC2** → **Target Groups**
2. 选择 `fyp-backend-tg`
3. 查看 **Health checks** 标签
4. 如果显示 "unhealthy"，查看失败原因

### 步骤 6: 如果健康检查失败

可能的原因和解决方案：

#### 问题 A: 健康检查路径不存在
**解决**: 已修复，现在使用 `/api/health/`

#### 问题 B: 后端服务未启动
**解决**: 
```bash
docker-compose -f docker-compose.prod.yml restart backend
docker-compose -f docker-compose.prod.yml logs -f backend
```

#### 问题 C: Migration 冲突
**解决**:
```bash
docker-compose -f docker-compose.prod.yml exec db mysql -uroot -pExam123456! -e "DROP DATABASE IF EXISTS db_exam; CREATE DATABASE db_exam CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
docker-compose -f docker-compose.prod.yml restart backend
```

#### 问题 D: gunicorn 未运行
**解决**: 检查 `requirements.txt` 是否包含 `gunicorn`，然后重新构建

### 步骤 7: 验证修复

```bash
# 1. 本地测试
curl http://localhost:8000/api/health/
curl http://localhost/

# 2. 从 ALB 测试（替换为你的 ALB DNS）
curl http://fyp-exam-system-alb-xxx.us-east-1.elb.amazonaws.com/api/health/
curl http://fyp-exam-system-alb-xxx.us-east-1.elb.amazonaws.com/
```

## 常见问题

### Q1: 容器一直重启

**检查日志**:
```bash
docker-compose -f docker-compose.prod.yml logs backend
```

**常见原因**:
- Migration 冲突 → 清空数据库
- gunicorn 未安装 → 检查 requirements.txt
- 端口被占用 → 检查端口监听

### Q2: 健康检查返回 500

**检查**:
```bash
# 查看详细错误
curl -v http://localhost:8000/api/health/

# 查看后端日志
docker-compose -f docker-compose.prod.yml logs backend
```

### Q3: 端口未监听

**检查**:
```bash
sudo ss -tlnp | grep -E ':(80|8000)'
```

如果端口未监听，检查容器是否正常运行。

## 如果问题仍然存在

1. **查看完整日志**:
   ```bash
   cat /var/log/user-data.log
   cat /tmp/deploy.log
   docker-compose -f docker-compose.prod.yml logs
   ```

2. **重新运行部署脚本**:
   ```bash
   cd /home/ubuntu/25FYP
   export DB_PASSWORD=Exam123456!
   export SECRET_KEY=$(openssl rand -base64 32)
   bash ./deploy/aws-deploy.sh
   ```

3. **检查安全组**:
   - 确保安全组允许来自 ALB 的流量（通常是 VPC CIDR）
   - 确保端口 80 和 8000 已开放
