## 📋 部署步骤

### 1. 设置环境变量（当前会话）

在 PowerShell 中运行：

```powershell
# 从 AWS Academy Learner Lab 获取凭证后，替换下面的占位符
$env:AWS_ACCESS_KEY_ID="YOUR_ACCESS_KEY_ID_HERE"
$env:AWS_SECRET_ACCESS_KEY="YOUR_SECRET_ACCESS_KEY_HERE"
$env:AWS_SESSION_TOKEN="YOUR_SESSION_TOKEN_HERE"
$env:AWS_DEFAULT_REGION="us-east-1"
```

### 2. 准备 SSH 密钥

**选项 A: 使用现有密钥**
```powershell
# 检查是否有现有密钥
Test-Path "$env:USERPROFILE\.ssh\id_rsa.pub"
```

**选项 B: 创建新密钥**
```powershell
# 生成 SSH 密钥对
ssh-keygen -t rsa -b 4096 -f "$env:USERPROFILE\.ssh\fyp-keypair"
```

### 3. 配置 Terraform 变量

`terraform.tfvars` 文件已创建，包含默认配置。

如果需要修改，编辑 `terraform.tfvars`：
- `key_pair_name`: 密钥对名称
- `public_key_path`: 公钥文件路径
- `enable_ha`: 是否启用高可用性（默认 true）

### 4. 预览部署计划

```powershell
cd infrastructure/terraform
terraform plan
```

这将显示将要创建的资源，**不会实际创建**。

### 5. 实际部署（关键演示步骤）

```powershell
terraform apply
```

输入 `yes` 确认后，Terraform 将：
- ✅ 创建 EC2 实例（2个，如果启用 HA）
- ✅ 创建应用负载均衡器
- ✅ 创建安全组
- ✅ 配置网络
- ✅ **完全不需要使用 Web Console！**

### 6. 查看输出

```powershell
terraform output
```

将显示：
- 实例 IP 地址
- 负载均衡器 URL
- SSH 连接命令

## ⚠️ 重要提示

### 1. 凭证有效期

AWS Academy Learner Lab 的凭证是**临时的**，通常有效期为：
- 1-4 小时（取决于 Learner Lab 设置）

如果凭证过期，需要：
1. 重新登录 AWS Academy
2. 启动新的 Learner Lab
3. 获取新的凭证
4. 更新环境变量

### 2. 成本控制

- 使用 `t2.micro` 免费层
- 默认不创建 RDS（使用 EC2 上的 MySQL）
- 部署完成后记得 `terraform destroy` 清理资源

### 3. 密钥对

确保 SSH 公钥文件存在，或使用 `public_key_content` 直接提供公钥内容。

## 🔧 常用命令

```powershell
# 查看计划
terraform plan

# 应用配置
terraform apply

# 查看状态
terraform show

# 查看输出
terraform output

# 销毁资源（清理）
terraform destroy

# 格式化代码
terraform fmt

# 验证配置
terraform validate
```
