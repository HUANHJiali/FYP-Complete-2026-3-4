# 修复 502 Bad Gateway 错误

## 问题分析

502 错误表示 ALB 无法连接到后端服务。可能的原因：

1. ✅ **ALB 配置错误** - 已修复：添加了前端目标组和路由规则
2. ✅ **GitHub 仓库名称错误** - 已修复：从 `FYP2025-12-27` 改为 `25FYP`
3. ⚠️ **Docker 容器未启动** - 需要检查
4. ⚠️ **健康检查失败** - 需要检查

## 已修复的问题

### 1. ALB 配置修复
- ✅ 添加了前端目标组（端口 80）
- ✅ 添加了后端目标组（端口 8000）
- ✅ 配置了路由规则：`/api/*` → 后端，其他 → 前端

### 2. GitHub 仓库名称修复
- ✅ 从 `FYP2025-12-27` 改为 `25FYP`

## 修复步骤

### 步骤 1: 重新部署 Terraform

```bash
cd infrastructure/terraform

# 应用修复后的配置
terraform apply
```

### 步骤 2: 等待部署完成（5-10分钟）

### 步骤 3: 检查 EC2 实例状态

SSH 到 EC2 实例：

```bash
ssh -i fyp-keypair.pem ubuntu@44.202.74.216
```

运行诊断脚本：

```bash
# 上传诊断脚本到 EC2
# 或者直接在 EC2 上运行以下命令

# 检查 Docker 容器
docker ps -a

# 检查端口监听
sudo netstat -tlnp | grep -E ':(80|8000)'

# 查看部署日志
tail -100 /tmp/deploy.log

# 查看 user-data 日志
tail -100 /var/log/user-data.log
```

### 步骤 4: 如果 Docker 容器未运行

手动启动服务：

```bash
cd /home/ubuntu/25FYP

# 检查 docker-compose 文件
ls -la docker-compose.prod.yml

# 启动服务
docker-compose -f docker-compose.prod.yml up -d

# 查看日志
docker-compose -f docker-compose.prod.yml logs -f
```

### 步骤 5: 检查 ALB 目标组健康状态

在 AWS Console 中：
1. 进入 **EC2** → **Target Groups**
2. 检查 `fyp-frontend-tg` 和 `fyp-backend-tg` 的健康状态
3. 如果显示 "unhealthy"，查看健康检查详情

## 常见问题

### Q1: Docker 容器启动失败

**检查日志：**
```bash
docker-compose -f docker-compose.prod.yml logs backend
docker-compose -f docker-compose.prod.yml logs frontend
```

**可能原因：**
- 数据库连接失败
- 端口被占用
- 内存不足

### Q2: 健康检查失败

**检查后端健康检查：**
```bash
curl http://localhost:8000/api/projects/all/
```

**检查前端健康检查：**
```bash
curl http://localhost/
```

### Q3: 端口未监听

**检查端口：**
```bash
sudo netstat -tlnp | grep -E ':(80|8000)'
```

如果端口未监听，检查 Docker 容器是否正常运行。
