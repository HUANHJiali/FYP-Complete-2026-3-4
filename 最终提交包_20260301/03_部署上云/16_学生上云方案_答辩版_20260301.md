# 学生上云方案（答辩版）2026-03-01

## 1. 方案目标
- 以最低复杂度完成“可上线、可演示、可复现”。
- 保持与本地一致的技术栈，降低迁移风险。
- 满足答辩关注点：稳定性、可运维性、可回滚。

## 2. 推荐架构（学生最优解）
- 云服务器：1 台 Ubuntu 22.04（2vCPU / 4GB / 50GB SSD）
- 部署方式：Docker Compose（三容器）
  - `frontend`
  - `backend`
  - `mysql`
- 入口：Nginx 反向代理
- HTTPS：Let's Encrypt（Certbot）

### 2.1 AWS Academy Learner Lab 对应方案（你当前可直接用）
- 账号类型：AWS Academy Learner Lab（预算 50 USD）
- 推荐资源（演示优先）：
  - EC2：`t3.small`（2 vCPU / 2GB）或 `t3.medium`（2 vCPU / 4GB，演示更稳）
  - EBS：`gp3` 30~50GB
  - 安全组：仅放行 `22/80/443`
- 部署方式不变：EC2 上跑 Docker Compose + Nginx

> 说明：Learner Lab 通常限制部分 AWS 服务权限，优先走“单机 EC2”路线，避免依赖复杂云服务（如 EKS、RDS、Route53）导致权限阻塞。

## 3. 为什么选这个方案
- 与当前项目结构完全一致：本地和云端同一套启动逻辑。
- 不需要上 Kubernetes，运维复杂度可控。
- 出问题可快速回滚到上一个 Git 提交。

## 4. 安全与网络最小基线
- 安全组仅开放：`22`、`80`、`443`
- SSH 仅白名单 IP
- 生产配置要求：
  - `DEBUG=False`
  - 强随机 `SECRET_KEY`
  - `ALLOWED_HOSTS` 仅域名/IP
  - 数据库密码改强口令

## 5. 上线步骤（可直接讲）
1. 准备云主机 + 域名解析 A 记录
2. 安装 Docker / Compose / Nginx
3. 拉取代码，配置 `.env.prod`
4. 启动容器：`docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --build`
5. 配置 Nginx 反代到 8080/8000
6. 申请 HTTPS 证书并开启自动续期
7. 验收：主页、健康检查、三角色登录、考试链路

## 6. 验收标准（答辩口径）
- 可访问：`https://你的域名`
- 健康检查：`/api/health/` 返回成功
- 三角色账号可登录（admin/teacher/student）
- 核心链路通过：学生提交 -> 教师发布 -> 学生查看结果

## 7. 故障预案
- 容器异常：`docker compose ... logs` 定位 + 重启服务
- 发布异常：切回上个稳定提交后重新 `up -d --build`
- 现场网络波动：优先展示自动化报告与本地可复现链路

## 8. 成本评估（学生可承受）
- 轻量云主机月成本：约 50~150 RMB（因厂商和活动浮动）
- 域名年成本：约 30~80 RMB
- HTTPS 证书：Let's Encrypt 免费

### 8.1 你的 50 USD 预算怎么花最稳
- 建议策略：
  1. 平时开发只在需要时启动实例（不用就 Stop）
  2. 答辩前 2~3 天连续开机做稳定性验证
  3. 答辩当天提前 1 小时开机预热
- 成本控制关键点：
  - **计算费来自实例运行时长**：不用就停机（Stop），不要长期 Running。
  - **存储费来自 EBS**：30GB 通常足够，避免开过大磁盘。
  - **公网流量**：演示流量通常很低，重点控制开机时长即可。

## 9. AWS Academy 实操步骤（精简）
1. 进入 Learner Lab，启动 Lab 环境。
2. 在 EC2 创建 Ubuntu 实例（`t3.small/t3.medium`）。
3. 绑定安全组：`22`（限本人 IP）、`80`、`443`。
4. SSH 登录实例，安装 Docker / Compose / Nginx。
5. 拉取项目并配置 `.env.prod`。
6. 执行生产启动命令：
   - `docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --build`
7. 验收：主页可访问、`/api/health/` 正常、三角色可登录、关键链路可走通。
8. 用完立即 `Stop` 实例（不是 Terminate），保留磁盘数据。

## 10. 答辩口径（针对 AWS Academy）
- 我使用 AWS Academy Learner Lab 的 50 USD 额度，采用单机 EC2 容器化部署方案。
- 该方案在权限受限环境下实现了最低复杂度上线，并通过健康检查、三角色登录与关键链路验证。
- 通过“按需开机 + 用后停机”的策略，有效控制预算并保障演示稳定性。

本项目采用“单机 Docker + Nginx + HTTPS”学生上云方案，以最小成本实现可部署、可验证、可回滚，保证答辩演示稳定可靠。
