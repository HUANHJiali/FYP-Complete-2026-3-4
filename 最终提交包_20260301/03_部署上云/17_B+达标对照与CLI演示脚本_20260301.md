## 2. 无控制台演示脚本（Windows PowerShell）

### 2.1 准备

```powershell
cd infrastructure/terraform

# 第一次使用：配置临时凭证
Copy-Item set-env.ps1.example set-env.ps1
# 编辑 set-env.ps1 填入 AWS Academy Learner Lab 的临时 AccessKey/Secret/SessionToken
.\set-env.ps1

# 确认身份（证明全程 CLI）
aws sts get-caller-identity
```

### 2.2 一键演示（推荐）

```powershell
cd infrastructure/terraform
.\demo-no-console.ps1 -SkipDestroy
```

说明：
- 脚本会自动执行 `terraform init / validate / plan / apply`。
- 自动输出 ALB DNS、前端 URL、后端 API URL、实例 ID。
- 自动发起 HTTP 健康检查，形成“自动化 + 可验证”证据。

### 2.3 手工分步演示（备选）

```powershell
cd infrastructure/terraform
terraform init
terraform validate
terraform plan -var="enable_ha=true" -out=tfplan
terraform apply -auto-approve tfplan

terraform output load_balancer_dns
terraform output backend_instance_ids
terraform output frontend_url
terraform output backend_api_url

# 健康检查
$url = terraform output -raw frontend_url
Invoke-WebRequest -Uri $url -UseBasicParsing
Invoke-WebRequest -Uri ($url.TrimEnd('/') + '/api/health/simple/') -UseBasicParsing
```

### 2.4 可选 HA 故障切换演示（加分）

```powershell
cd infrastructure/terraform
$ids = terraform output -json backend_instance_ids | ConvertFrom-Json
aws ec2 stop-instances --instance-ids $ids[0]
aws ec2 wait instance-stopped --instance-ids $ids[0]

$url = terraform output -raw frontend_url
Invoke-WebRequest -Uri $url -UseBasicParsing
```

若页面仍可访问，可说明 ALB 将流量转发到另一台健康实例，展示了 HA 能力。

### 2.5 演示结束回收资源（预算控制）

```powershell
cd infrastructure/terraform
terraform destroy -auto-approve -var="enable_ha=true"
```

## 3. 答辩讲解话术（30秒）

“本项目云端部分使用 Terraform 进行基础设施即代码部署，命令行一键创建 2 台 EC2、ALB、目标组和监听规则，自动完成应用部署。演示时我不使用 AWS 控制台，只用 CLI 执行 `init/plan/apply`，并通过 URL 与健康检查命令验证服务可用。最后执行 `terraform destroy` 回收资源，体现自动化、可复现和成本可控。”

## 4. 现场风险与兜底

- AWS Academy 临时凭证过期：提前 10 分钟刷新并 `aws sts get-caller-identity` 预检。
- 资源配额不足：优先保证 `enable_ha=true` + `instance_type=t2.micro` 演示最小闭环。
- 构建时间偏长：先展示 `terraform output` 与健康检查，再补充业务页面操作。
