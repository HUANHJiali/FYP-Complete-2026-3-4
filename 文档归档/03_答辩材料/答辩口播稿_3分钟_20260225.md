# 答辩口播稿（3分钟版）- 2026-02-25

## 开场（约20秒）
各位老师好，我的FYP项目是“智能在线考试系统”，面向管理员、教师和学生三类角色。系统采用 Django + Vue3 + MySQL 架构，并接入 AI 评分与 AI 题目生成功能。今天我重点汇报三点：系统是否可运行、功能是否闭环、证据是否可审计。

## 第一部分：系统可运行（约45秒）
首先是可运行性。项目支持 Docker 一键启动，当前版本已实现自动构建、自动就绪等待、健康检查和登录自检。也就是说，启动后不仅容器起来了，还会自动验证后端健康接口与登录接口是否正常。

对应证据在：
- `docker-compose.yml`
- `docker-start.bat` / `docker-start.sh`
- `最终可交付回归清单_20260225.md` 第9节

这部分结论是：当前提交版本具备稳定演示和复现能力。

## 第二部分：功能闭环（约60秒）
第二是功能闭环。系统已完成三角色关键链路：
- 管理员：消息、任务、统计、日志管理。
- 教师：考试审核与成绩发布。
- 学生：任务练习、错题复习、考试记录与消息中心。

最新一轮回归聚焦高风险写操作接口，结果是链路级全通过（code=0），并且问题都形成了“发现-修复-复测”的闭环记录。

对应证据在：
- `最终可交付回归清单_20260225.md` 第3-6节
- `Kickoff达标证据索引_20260225.md`

第三是工程化和质量。我们建立了周志链路、团队贡献矩阵和KPI评估模板：
