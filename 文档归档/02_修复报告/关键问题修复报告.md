## 🔧 问题 1：工具栏遮挡内容

### 问题描述
固定定位的顶部工具栏（60px高）遮挡了页面内容。

### 根本原因
```css
.fater-answer-toolbar {
    position: fixed;
    top: 0;
    height: 60px;
}
```
- 工具栏使用 `position: fixed` 固定在顶部
- 主容器没有 `padding-top` 避开工具栏
- 导致页面顶部 60px 内容被遮挡

### 修复方案
在 `answer.vue` 的 `<style scoped>` 中添加：
```css
/* 主容器顶部内边距 - 避免被工具栏遮挡 */
.fater-answer-panle {
    padding-top: 60px;
}
```

### 修复位置
- **文件**: `source/client/src/views/pages/answer.vue`
- **行数**: 第 186 行后添加

### 修复效果
- ✅ 页面内容不再被工具栏遮挡
- ✅ 布局正常显示
- ✅ 用户体验提升

## 🔧 问题 2：v-model 响应性失效

### 问题描述
答题时，用户选择的答案可能不会正确保存到列表中。

### 根本原因
```javascript
// ❌ 错误做法
getItem(id, no) {
    getPractiseInfo(id).then(resp => {
        this.practise = {  // 创建新对象，导致 v-model 失去响应性
            id: resp.data.id,
            name: resp.data.name,
            answer: temp.answer,
            // ...
        };
    });
}
```

**问题分析**:
1. `this.practise = { ... }` 创建了一个全新的对象
2. v-model 在模板中绑定的是**旧对象**的 `answer` 属性
3. 切换题目时，`practise` 变量指向新对象
4. 但 v-model 仍然监听旧对象的 `answer` 属性
5. 导致用户输入无法同步到列表中的数据

### 修复方案
```javascript
// ✅ 正确做法
getItem(id, no) {
    getPractiseInfo(id).then(resp => {
        // 使用 Object.assign 或直接赋值，保持对象引用不变
        Object.assign(this.practise, {
            id: resp.data.id,
            name: resp.data.name,
            answer: temp.answer,
            // ...
        });
    });
}
```

**优势**:
- ✅ 保持对象引用不变
- ✅ v-model 继续监听同一个对象的属性
- ✅ 用户输入正确同步到列表
- ✅ 响应式系统正常工作

## 🎯 Vue 响应式原理说明

### 为什么会出现这个问题？

Vue 2 的响应式系统使用 `Object.defineProperty` 来监听对象属性的变化。

```javascript
// Vue 在初始化时会这样做
Object.defineProperty(vm.practise, 'answer', {
    get() { /* ... */ },
    set(newValue) { /* ... */ }
});
```

**当我们执行 `this.practise = { ... }` 时**：
1. 创建了一个新对象
2. Vue 对新对象的属性也建立了响应式监听
3. 但是！v-model 在模板中编译时绑定的是**旧对象**的 getter/setter
4. 用户的输入仍然会触发旧对象的 setter
5. 但列表中的 `practise` 变量已经指向新对象
6. 所以输入无法同步到列表

**当我们使用 `Object.assign(this.practise, { ... })` 时**：
1. 对象引用保持不变
2. 只更新属性的值
3. v-model 绑定的 getter/setter 仍然有效
4. 用户输入正确同步到列表
