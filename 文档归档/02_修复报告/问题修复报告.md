```python
# comm/AIUtils.py (新增)
def test_ai_connection() -> bool:
    """
    测试 AI 服务连接

    Returns:
        True if AI service is available, False otherwise
    """
    try:
        import os
        from django.conf import settings

        # 检查 API Key 是否配置
        api_key = (
            os.getenv('ZHIPUAI_API_KEY')
            or os.getenv('OPENAI_API_KEY')
            or getattr(settings, 'OPENAI_API_KEY', '')
        )

        if not api_key or api_key == '':
            print("AI 服务未配置：API Key 为空")
            return False

        # 尝试发送一个简单的测试请求
        ai_utils = AIUtils()

        # 发送测试请求
        url = f"{ai_utils.base_url.rstrip('/')}/chat/completions"
        headers = {
            'Authorization': f'Bearer {ai_utils.api_key}',
            'Content-Type': 'application/json',
        }
        payload = {
            'model': ai_utils.model,
            'messages': [{"role": "user", "content": "Hi"}],
            'max_tokens': 10,
        }

        import requests
        resp = requests.post(url, headers=headers, json=payload, timeout=10)

        if resp.status_code == 200:
            print("AI 服务连接成功")
            return True
        elif resp.status_code == 401:
            print("AI 服务认证失败：API Key 无效")
            return False
        else:
            print(f"AI 服务返回错误：{resp.status_code}")
            return False

    except Exception as e:
        print(f"AI 服务测试失败：{str(e)}")
        return False
```

### 修复结果
✅ **导入错误已修复**
- 相对导入改为绝对导入
- 添加了 `test_ai_connection()` 函数

⚠️ **AI 服务仍显示 unhealthy**
- 原因：可能未配置 API Key 或 API Key 无效
- 状态：代码已修复，需要配置有效的 API Key

## 🟡 问题 2: 错题本路由问题

### 问题描述
```
GET /api/wrongquestions/page/ → 404 Not Found
```

### 根本原因
❌ **不是路由问题！** 路由配置完全正确：
```python
path('wrongquestions/<str:module>/', WrongQuestionsView.as_view())
```

✅ **真实原因**：视图使用的模块名不匹配
- 错误调用：`/api/wrongquestions/page/`
- 正确调用：`/api/wrongquestions/getPageInfos/`

### 视图模块列表
```python
# WrongQuestionsView 支持的模块：
def get(self, request, module, *args, **kwargs):
    if module == 'info':
        return self.get_info(request)
    elif module == 'getPageInfos':          # ✅ 正确
        return self.get_page_infos(request)
    elif module == 'getStudentWrongQuestions':
        return self.get_student_wrong_questions(request)
    elif module == 'getWrongQuestionDetail':
        return self.get_wrong_question_detail(request)
    elif module == 'getReviewHistory':
        return self.get_review_history(request)
    else:
        return BaseView.error('请求地址不存在')
```

### 测试结果
```bash

curl http://localhost:8000/api/wrongquestions/page/
# 结果：404 Not Found

# ✅ 正确调用

# 结果：200 OK（接口正常工作）
```

### 修复方案
✅ **无需修复代码** - 接口工作正常

**建议**：
1. 前端调用时使用正确的模块名：`getPageInfos` 而不是 `page`
2. 或者在视图中添加 `page` 模块作为 `getPageInfos` 的别名
