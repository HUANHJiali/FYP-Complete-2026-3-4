### P1 - 重要问题（2个）

#### 1. ⚠️ 答题页面 v-model 绑定可能失效

**问题描述**:
```vue
<!-- answer.vue -->
<RadioGroup v-if="practise.type==0" v-model="practise.answer" vertical>
  <Radio v-for="(item, index) in practise.options"
          :key="index" :label="item.id">{{ item.name }}</Radio>
</RadioGroup>
```

**潜在风险**:
- `practise` 对象在 `getItem()` 方法中被完全替换
- v-model 可能在对象替换时失去响应性
- 答案可能不会自动保存到列表中

**影响**: 用户选择的答案可能丢失

**复现步骤**:
1. 选择一道选择题的答案
2. 切换到下一题
3. 切回上一题
4. 答案可能消失

**修复方案**:
```javascript
// 不要完全替换 practsie 对象，而是更新属性
getItem(id, no) {
  getPractiseInfo(id).then(resp => {
    // 更新而不是替换
    this.practise.id = resp.data.id;
    this.practise.name = resp.data.name;
    this.practise.type = resp.data.type;
    this.practise.options = resp.data.options;
    // 答案从列表中恢复，保留响应性
  });
}
```

**优先级**: 🔴 高
**难度**: ⭐⭐⭐
**预计时间**: 1小时

### P2 - 中等问题（3个）

#### 3. ⚠️ 键盘快捷键与输入框冲突

**问题描述**:
```javascript
handleKeyPress(e) {
    // 在输入框中，不响应快捷键
    if (e.target.tagName === 'INPUT' || e.target.tagName === 'TEXTAREA') {
        if (!(e.ctrlKey && e.key === 's')) {
            return;
        }
    }
    // 方向键切换题目
    if (e.key === 'ArrowLeft') this.previousQuestion();
}
```

**潜在风险**:
- 在编程题（textarea）中，方向键用于移动光标
- 当前逻辑会阻止使用方向键移动光标

**影响**: 编程题输入体验差

**修复方案**:
```javascript
handleKeyPress(e) {
    // 在输入框中，只响应 Ctrl+S，其他都不响应
    if (e.target.tagName === 'INPUT' || e.target.tagName === 'TEXTAREA') {
        if (e.ctrlKey && e.key === 's') {
            e.preventDefault();
            this.saveAnswer(this.practise);
            this.$Message.success('答案已保存');
        }
        return; // 输入框中不响应其他快捷键
    }

    // 只在没有聚焦输入框时响应方向键
    if (e.key === 'ArrowLeft') {
        e.preventDefault();
        this.previousQuestion();
    }
    // ...
}
```

**优先级**: 🟡 中
**难度**: ⭐⭐
**预计时间**: 30分钟

#### 4. ⚠️ 首次引导与系统冲突

**问题描述**:
```javascript
checkWelcomeGuide() {
    const hasShown = localStorage.getItem('fyp_welcome_guide_shown');
    if (!hasShown) {
        setTimeout(() => {
            this.showWelcomeGuide = true;
        }, 1000);
    }
}
```

**潜在风险**:
- 使用 localStorage 而不是用户级设置
- 换设备或清理浏览器后再次显示
- 没有与用户类型关联（管理员/教师/学生）

**影响**: 体验不一致

**修复方案**:
- 添加到用户设置中（后端存储）
- 根据用户角色显示不同的引导内容

**优先级**: 🟡 低
**难度**: ⭐⭐⭐
**预计时间**: 2小时

#### 5. ⚠️ 空状态只在题目未加载时显示

**问题描述**:
```vue
<div v-else class="empty-state">
    <!-- 只在 practsie 为空时显示 -->
</div>
```

**潜在风险**:
- 如果题目列表为空但 practsie 有值，不会显示空状态
- 用户看到的是空白界面

**影响**: 用户体验差

**修复方案**:
```vue
<div v-if="totalCount === 0" class="empty-state">
    <!-- 当总题目数为 0 时显示 -->
</div>
```

**优先级**: 🟡 低
**难度**: ⭐
**预计时间**: 15分钟
